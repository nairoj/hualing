# 数值系统设计文档（简版 MVP）

## 1. 概念
本项目数值系统以“道具与伙伴驱动成长”为核心，不采用角色等级升级。数值目标是让玩家在 12 分钟探索周期内做出清晰的战斗和撤离决策。

## 2. 设计目标
1. 保持战斗反馈直接，便于双人小团队快速迭代。
2. 让三个固定伙伴形成明确功能分工。
3. 支撑“白天推进、夜晚高压”的时间策略。

## 3. 数值总则（MVP）
- 难度档：当前仅 `标准`。
- 成长方式：通过伙伴与道具，不走角色等级。
- 昼夜压力：夜晚刷怪倍率 3.0，黄昏线性过渡。

## 6. 战斗数值规则
### 6.1 伤害公式（简版）
`最终伤害 = max(1, 攻击力 * 技能倍率 - 防御力)`

### 6.2 单位行为
- 玩家：固定近战。
- 锋锋（剑灵）：自动跟随，自动攻击，就近选敌。
- Buff共享：玩家与剑灵共享蜡烛增伤。

### 6.3 时间压力数值
| 时间段 | 刷怪倍率 | 说明 |
|---|---:|---|
| 白天（06:00-18:00） | 1.0 | 主推进阶段 |
| 黄昏（18:00-20:00） | 1.0 -> 3.0 | 线性提高压力 |
| 夜晚（20:00-06:00） | 3.0 | 高风险阶段 |
