# 化灵系统设计文档（MVP）

## 1. 概念
“化灵”是本项目的核心机制：将指定物品转化为可同行的伙伴。伙伴会继承原物品功能，并在探索中提供更强的战斗或辅助价值。

## 2. 设计目标
1. 让玩家在进入地城前进行明确的编队决策。
2. 通过化灵对象取舍，制造“家园功能 vs 闯关收益”的选择。
3. 在 MVP 阶段先验证固定 3 伙伴的循环乐趣，不引入复杂随机。

## 3. MVP 范围
- 可化灵对象：`柜子 / 蜡烛 / 小狗`。
- 化灵成功率：100%。
- 化灵失败：不启用。
- 性格与冲突：不启用。
- 相性系统：不启用。

## 4. 核心规则
### 4.1 化灵前置条件
- 玩家持有化灵散。
- 目标属于 MVP 白名单物品。
- 同一物品仅可化灵一次。

### 4.2 化灵流程
1. 选择目标物品。
2. 消耗化灵散并播放特效。
3. 原物品从原位置移除。
4. 生成对应伙伴并加入可编队列表。

### 4.3 唯一性与持久化
- 一个物品化灵后即转为伙伴实体，不可再次化灵。
- 已化灵物品不可召回为原家具（MVP）。
- 夜晚休息时，化灵伙伴在屋外帐篷驻留。

### 4.4 死亡与刷新
- 伙伴在探索中死亡后当日消散。
- 第 2 天 06:00 在家园外帐篷刷新。

## 5. 与核心循环的关系
- 进入洞穴前进行伙伴编队选择。
- 白天优先推进，夜晚风险增加时更依赖伙伴功能保命与撤离。

## 6. 数据字段建议（程序/表格）
| 字段 | 类型 | 说明 |
|---|---|---|
| partner_id | string | 伙伴唯一 ID |
| source_item_id | string | 原始物品 ID |
| partner_name | string | 伙伴名 |
| role | enum | 输出/辅助/功能 |
| active_skill_id | string | 主动技能 ID |
| passive_effect_id | string | 被动效果 ID |
| summon_cost | int | 化灵消耗 |
| can_revert | bool | 是否可还原（MVP=false） |
| respawn_time | string | 刷新时间（06:00） |
| mvp_enabled | bool | MVP 是否启用 |
